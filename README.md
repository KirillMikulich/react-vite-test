# Setklapp

Тестовое задание на `React + TypeScript`.

## Что есть

- таблица с колонками: `Имя`, `Дата`, `Значение`, `Действия`
- добавление записи через модалку
- редактирование записи
- удаление записи
- валидация в форме
- поиск по таблице
- сортировка по имени, дате и числу
- светлая и тёмная тема
- сохранение данных в `localStorage`
- русский язык в интерфейсе

## Стек

- `React`
- `TypeScript`
- `Vite`
- `Ant Design`
- `Zustand`
- `SCSS Modules`
- `dayjs`

## Структура проекта

Структуру сделал как упрощённую кальку на `FSD`, без лишних слоёв. Для такого тестового не видел смысла усложнять архитектуру, поэтому лишнее убрал.

```text
src/
  app/         запуск приложения, провайдеры
  components/  таблица, модалка, тулбар, переключатель темы
  hooks/       хуки
  models/      типы, селекторы, утилиты, тема
  pages/       страницы
  state/       zustand store
  styles/      глобальные стили
```

Основные файлы:

- `src/pages/Records/index.tsx` — сборка страницы
- `src/components/Table/index.tsx` — таблица
- `src/components/RecordModal/index.tsx` — модалка добавления и редактирования
- `src/components/Toolbar/index.tsx` — поиск и кнопка добавления
- `src/state/records-store.ts` — записи и операции с ними
- `src/state/app-settings-store.ts` — тема приложения

## Пояснения

`records-store` сделал глобальным, потому что там лежат сами данные таблицы: список записей, добавление, редактирование и удаление.

Состояние модалки вынес отдельно, чтобы не смешивать данные таблицы и UI-состояние.

Поиск оставил локально на странице, потому что тут нет API и не было задачи синхронизировать его с URL.

Тему и данные сохраняю через `persist`, чтобы после перезагрузки всё не сбрасывалось.
